# 模型配置更新流程

## 目的

确保每次模型配置变更（新增模型、调价、参数调整、端点变更）都有明确的来源记录和验证步骤，防止误改和遗漏。

## 变更类型

| 类型 | 影响范围 | 必要步骤 |
|------|----------|----------|
| 新增模型 | MODELS, pricing-snapshot, billing calculate, 前端 model card | 1-5 全部 |
| 调价 | MODELS, pricing-snapshot, billing | 1, 2, 3, 5 |
| 参数调整 | MODELS, model-validator, 前端参数表 | 1, 2, 4, 5 |
| 端点变更 | MODELS | 1, 2, 5 |
| 删除模型 | MODELS, pricing-snapshot | 1, 2, 3, 5 |

## 步骤

### 1. 记录来源

在 `packages/provider/test/pricing-snapshot.test.ts` 头部注释中记录来源链接和更新日期。

来源链接必须是阿里云百炼 DashScope 官方定价页或 API 文档页：
- 文本模型: https://help.aliyun.com/zh/model-studio/getting-started/models
- 图像模型: https://help.aliyun.com/zh/model-studio/developer-reference/image-generation-api
- 视频模型: https://help.aliyun.com/zh/model-studio/developer-reference/video-generation-api

`docs/bailian` 中的资料文档只作为参考，不直接把其中的示例代码复制进业务代码。

### 2. 更新快照

修改 `packages/provider/test/pricing-snapshot.test.ts` 中的 `EXPECTED_PRICING`，与新定价一致。

如果新增模型，必须在快照中增加对应条目，否则反向检查会报错。

如果删除模型，必须从快照中移除对应条目，否则正向检查会报错。

### 3. 运行测试

```bash
# 定价快照测试
bun test packages/provider/test/pricing-snapshot.test.ts

# 模型配置一致性测试
bun test packages/provider/test/model-configs.test.ts

# 模型参数校验测试
bun test packages/provider/test/model-validator.test.ts

# 计费逻辑测试
bun test packages/billing/test/calculate.test.ts

# 全量测试
bun run test
```

所有测试必须通过。

### 4. 更新相关代码

- 参数调整 → 同步更新 `packages/provider/src/model-validator.ts` 的校验逻辑
- 新增模型 → 同步更新前端模型卡片展示（`apps/client/src/pages/Workspace.tsx` 等）
- 新增视频模型 → 同步更新 Worker 轮询逻辑（`apps/worker`）

### 5. Commit message

commit message 必须注明变更类型和来源：

```
feat: add wan2.7-t2v model — 来源: https://help.aliyun.com/zh/...
fix: adjust qwen-max pricing from 240→300 inputPriceCents — 来源: https://help.aliyun.com/zh/...
```

## 价格配置与请求参数配置的关系

当前 `model-configs.ts` 中 pricing 和 parameters 在同一个 `ModelConfig` 对象中。两者逻辑独立：

- **pricing**: 影响计费计算 (`@excuse/billing`) 和前端展示
- **parameters + inputMapping**: 影响请求构建 (`@excuse/provider`) 和前端参数表

审计定价变更时只需关注 `pricing` 字段和 `pricing-snapshot.test.ts`。
审计参数变更时只需关注 `parameters` 和 `inputMapping` 字段和 `model-configs.test.ts`。

两者共享 `MODELS` 数据源，但可独立审计和验证。
