## 一、Canvas CRUD 补全（高优先级）

### 1.1 项目标题/故事文本不可编辑 ✅ 已完成

**问题：** 用户创建项目后无法修改 `title` 和 `storyText`，写错只能删项目重建。

**位置：**
- 路由层：`apps/server/src/routes/canvas.ts` — 没有 `PUT/PATCH /projects/:projectId` 通用更新端点
- Service 层：`apps/server/src/modules/canvas/service.ts` — 没有 `updateProjectProperties()` 函数
- DB 层：`packages/db/src/repositories/canvas-projects.repo.ts:32-42` — `updateCanvasProject()` 的类型约束只允许 `title | status | analysisJson | modelPreferencesJson | canvasLayout`，**不包含 `storyText`**

**解决方案：**
1. `packages/db/src/repositories/canvas-projects.repo.ts` — 扩展 `updateCanvasProject` 的 `values` 类型，加入 `storyText`
2. `apps/server/src/modules/canvas/service.ts` — 新增 `updateProjectProperties(projectId, { title?, storyText? })`
3. `apps/server/src/routes/canvas.ts` — 新增路由：
   ```
   PATCH /api/canvas/projects/:projectId
   body: { title?: string, storyText?: string }
   ```
4. 前端 `NodeDetailPanel.tsx` 或 `CanvasEditor.tsx` — 给 Analysis 节点加编辑按钮，可改 title/storyText

### 4.2 计费浮点数运算

**问题：** 所有金额计算使用 `roundTo4()` 浮点数（4 位小数），未转为整数分。浮点运算可能导致金额偏差（如 0.1 + 0.2 !== 0.3）。

**位置：**
- `packages/billing/src/calculate.ts:85-87` — `roundTo4()` 浮点四舍五入
- `packages/billing/src/calculate.ts:27-29` — token 计费 `(tokens / 1_000_000) * price` 浮点乘法
- `packages/billing/src/statistics.ts` — aggregateStatistics 也用浮点

**解决方案：**
1. 所有价格配置改为**整数分**（CNY cents）：`inputPrice: 0.002` → `inputPriceCents: 200`
2. `calculateCost()` 内部运算全部用整数：`(tokens * inputPriceCents) / 1_000_000`
3. 最终返回值同时包含 `totalPriceCents` 和 `totalPrice`（cents / 100 转元，展示用）
4. `packages/provider/src/model-configs.ts` — 所有 pricing 字段改为 cents
5. 前端展示层：`totalPriceCents / 100` → 显示金额

---

### 4.3 getCostRecords 全表扫描 + JS 过滤

**问题：** `getCostRecords()` 拉所有 cost 非空的记录到内存，再用 JS 过滤 totalPrice 是否为数字。无分页、无日期范围。

**位置：**
- `packages/db/src/repositories/generation-records.repo.ts:131-144`

**解决方案：**
1. SQL 层直接聚合：`SELECT category, model, SUM((cost->>'totalPrice')::numeric) as total, ...` 而不是拉全表
2. 支持日期范围：`WHERE created_at BETWEEN $1 AND $2`
3. 支持 accountId 过滤
4. 如果不想依赖 JSONB 内部查询，可以新增 `total_price_cents` 整数列（见 4.2），直接 SQL SUM

## 七、Worker 可靠性（中优先级）

### 7.1 Worker 优雅退出不等待当前任务完成 ✅ 已完成

**问题：** 收到 SIGINT/SIGTERM 时只设 `running = false`，不等待当前 `processTask` 完成。如果任务正在下载视频文件，可能导致 shot 状态停在 `generating` 但视频 URL 永远不会写入。

**解决方案已实施：**
1. 信号处理改为 async handler：设 `running = false` 后 `await currentTaskPromise`
2. `currentTaskPromise` 变量跟踪当前正在处理的任务
3. 设置 30 秒超时：超时后 `process.exit(1)` 强制退出
4. 正常完成后 `process.exit(0)`
